# Service template

Clone this folder for every new project. It's the one piece every project shares:
a FastAPI app with a `/health` and `/run` route, wrapped in a Dockerfile.

## What to change per project
- `app/main.py` — replace the body of `/run` with your actual logic, and add
  more routes if needed. Keep `/health` as-is; the deployment layer depends on it.
- `requirements.txt` — add your project's dependencies.
- The `title=` in `FastAPI(...)` — rename to your project's name.

## What NOT to change (unless the whole platform needs it)
- `Dockerfile` — same shape across all projects. Keeps builds and AWS
  deployment consistent.
- The `/health` route's path and response shape.

## Running locally

\`\`\`bash
docker compose up --build
\`\`\`

This builds the image, starts the container, and mounts `app/` as a volume
with `--reload` so code changes show up without rebuilding. Visit:

- http://localhost:8000/docs — interactive API docs (auto-generated by FastAPI)
- http://localhost:8000/health — should return `{"status": "ok"}`

## Running without compose

\`\`\`bash
docker build -t my-project .
docker run -p 8000:8000 my-project
\`\`\`

This skips the volume mount and `--reload` — useful for testing the image the
way it'll actually run in production (including on AWS later), since compose's
dev mode hides packaging mistakes that only show up in a clean build.
